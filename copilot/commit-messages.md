# Commit Message Instructions

To generate commit messages for this project, follow these guidelines:

- Prefix the subject with a type indicator, such as "feat:" for new features, "fix:" for bug fixes, "hotfix:" for urgent fixes.
- Use imperative mood for the subject (e.g., "Add feature" instead of "Added feature").
- Keep the subject line under 50 characters.
- Use the body to provide more details on what changed and why, if needed.
- Reference issue numbers or pull requests when applicable (e.g., "Fixes #123").
- Separate subject from body with a blank line.
- Wrap lines at 72 characters.

## Recommended Symbols/Icons

- feat: 🚀 (new features)
- fix: 🐛 (bug fixes)
- hotfix: 🚑 (urgent fixes)
- docs: 📚 (documentation)
- refactor: ♻️ (code refactoring)
- test: 🧪 (adding tests)
- chore: 🔧 (maintenance tasks)
